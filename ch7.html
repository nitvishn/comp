Programming Languages

  • A computer program is a list of instructions that enable a computer to perform a specific task.
  • A program is written in a programming language.

Languages

High-level languages

  • Designed with programmers in mind
  • Language used is similar to human language
  • Problem-oriented
  • No knowledge of the hardware and instruction set of the computer is needed
  • Easier to:
    • read and understand
    • write quickly
    • debug when in use
    • maintain when in use

Low-level languages

  • Relate to the specific architecture and hardware of a particular type of computer.
  • Types of low-level languages:
    • Assembly languages
      • Few programmers write in this, as the language is very arcane and hard to understand.
      • An assembler is needed to translate this to machine code.
      • Programmers write in this for the following reasons:
        • to make use of special hardware
        • to make use of special machine-dependant instructions
        • to write code that doesn’t take up space in primary memory
        • to write code that performs a task very quickly.
    • Machine code
      • This is a computer programming language consisting of binary or hexadecimal instructions which a computer can respond to directly.
      • These are executed directly by the computer’s CPU.
      • This code is in binary, but is usually shown in hexadecimal.

Translators

Compilers

  • Translates a high-level program to machine code.
  • An object file is produced, which can then be processed and executed by the computer.
  • This object file can be executed as many times as needed.
  • Compiled programs (the object file) can be used without the compiler.
  • If a program contains a syntax error, a comprehensive list of all syntax errors is produced.
  • Usually distributed for general use.

Interpreters

  • Executes a high-level language program one statement at a time.
  • No executable file of machine code is produced.
  • Interpreted programs cannot be used without the interpreter.
  • If the program contains a syntax error, interpreters will stop execution as soon as they encounter the first syntax error.
  • Often used when a program is being developed.

Assemblers

  • Translates a low-level language program into machine code.
  • An executable file of machine code is produced.
  • One low-level language statement is usually translated into one machine code instruction.
  • Assembled programs can be used without the assembler.
  • An assembled program is usually distributed for general use.

Errors

  • A syntax error is where a program statement does not obey the grammatical rules of the programming language.
    • There are two types of syntax errors:
      1. When commands not present in the programming language are used. In English, this would be: ”I am the glabglabgugalab".
      2. When commands present in the programming language are used, but in a way that doesn’t make grammatical sense. For example, in English: “I are inevitable."
    • Syntax errors are usually caught by a compiler or interpreter.
  • A logic error is when a program does not do what it was intended to do.
  • A runtime error is when a program encounters issues with some operations during runtime. For example, a runtime error is produced when a preposterous statement like ”I love you” + 3000 is found. This could be due to wrong types of input, for example.
