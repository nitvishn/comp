Practical problem-solving and programming

Algorithms

  • An algorithm is a set of instructions that have to be performed in order to carry out a task.
  • This can be detailed through a flowchart or pseudocode.
  • A trace table can be used to inspect the function of an algorithm closely.

Test Data

  • Normal test data: the algorithm should accept this value.
  • Erroneous/abnormal test data: the algorithm should reject this value.
  • Boundary data: data to establish where the largest and smallest accepted values occur.

Validation and verification

Validation.

  • Automatic, done by an algorithm.
  • Checks whether or not data inputs are reasonable/accurate.
  • Types:
    1. Range checks.
    2. Length checks.
    3. Type checks.
    4. Character checks.
    5. Format checks.
    6. Presence checks.
    7. Check digits.

Verification.

  • Checks whether:
    1. Data has been copied to the computer accurately.
    2. Data has been transferred from one computer to another accurately.
  • Types:
    1. Double entry.
    2. Visual check.
    3. Parity check.
    4. Checksum.

Data Types

  1. Int - integer
  2. Real or Float - this is a decimal value.
  3. Char - this is just one character, like 'a' or '&'.
  4. String - this is a list of characters, like ‚ÄùUncle Barney juggles eggplants for a living".
  5. Bool - has two values, True or False. Can also be Yes or No.
  6. Date - represents time on any scale, like seconds, minutes, days, years, etc.

Data Structures

  1. Array
    • Is a finite list of variables, each of which hold data that is of a type given above.
    • This can be indexed - Names[10] will give you the 10th element of the array Names.
    • The size of the array has to be given while declaring it.
    • Arrays cannot change size during the runtime of the program, unless a new array is created and it is reassigned to that name.
  2. Variable
    • Can have any of the above data types.
    • The value can change during the runtime of the program.
  3. Constant
    • Can have any of the above data types.
    • This value cannot change after it is declared.
